# Agents Working Rules

이 저장소에서 Codex 에이전트는 아래 규칙을 기본으로 따른다.

## 1) 완료 기준
- 사용자 요청으로 코드/문서/설정이 변경되면, 작업 완료 시점에 반드시 Git 커밋을 만든다.
- 커밋 후 반드시 원격 저장소(`origin`)로 푸시한다.

## 2) 기본 절차
1. 변경사항 검증(가능한 범위에서 빌드/테스트 실행)
2. `git add -A`
3. 명확한 커밋 메시지로 `git commit`
4. 현재 작업 브랜치에서 `git push origin <branch>`

## 3) 예외 처리
- 푸시 실패(인증/네트워크/권한) 시, 실패 원인과 재시도 결과를 사용자에게 즉시 보고한다.
- 충돌/보호 브랜치 정책으로 직접 푸시가 불가하면, 푸시 가능한 대안 브랜치 생성 후 푸시하고 안내한다.
